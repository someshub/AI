11 Things Senior Agentic AI Engineers

Semantic Kernel & Orchestration Internals
Not just "use SK". They test deeper knowledge:
  • StepwisePlanner vs HandlebarsPlanner trade-offs
  • DI Lifetimes: Per-request Kernel vs Singleton memory leaks
  • Why AutoInvoke=false is critical for HIL approval
  • Deadlock risks: Multi-agent infinite loops & MaxIterations

🔍
RAG Architecture at Scale
Understand trade-offs beyond vector search:
  • Chunking: Semantic vs Fixed. Tables & headings handling
  • Hybrid Search: Vector + BM25 + Semantic Ranker = 98% Recall@3
  • Eval with RAGAS: Faithfulness, Answer Relevance metrics
  • Why In-Memory fails: Need Cosmos DB/AI Search for 100M+ scale
3
🤝
Multi-Agent Patterns
Move beyond AutoGen basics:
  • GroupChat vs GraphChat: Broadcast vs DAG. When to use
  • Termination: MaxMessage, TextMention, FunctionCall to stop loops
  • Judge/Critic Agents: Reflexion pattern. 3x cost but 40% quality up
  • Cost explosion: 4 agents × 3 turns = 12 LLM calls. Use SLM for critics
4
🎙️
Voice AI & Real-time Latency
Identify & fix <800ms pipeline issues:
  • Twilio STT Streaming: Use Recognizing event, not Recognized
  • Barge-in: StopSynthesisAsync on user interrupt with CancellationToken
  • TTS Sentence Buffering: Stream tokens, speak on `.` or `?`
  • Network Jitter: Same-region ACA + OpenAI + Cosmos DB
5
🛡️
Guardrails & Responsible AI
Validate and enforce at 3 layers:
  • PII Masking: Presidio + Azure Language. Input + Output + Embedding filters
  • Prompt Injection: Content Safety Jailbreak + Instruction Defense
  • DPDP Compliance: Cosmos TTL, Purview audit, Data minimization
  • HIL for >1L txn: Durable Functions + Teams Adaptive Card approval

📊
LLMOps & Evaluation
Move beyond "it works":
  • Prompt Versioning: Prompt Flow + Git. Track perf per version
  • Groundedness Scoring: LLM-as-Judge. <0.8 = fallback to human
  • Latency P95: Track dependency_duration in App Insights
  • Cost per 1k req: Token counter + Meter API. Alert if >₹10
7
☁️
Azure OpenAI Production
Optimize TPM/RPM like a pro:
  • Quota Management: Multi-region. 10k RPM × 3 = 30k capacity
  • Semantic Caching: Redis + Embedding. 60% hit = 60% cost save
  • Batch API: 50% cheaper for offline embedding jobs
  • SLM Routing: Phi-3-mini for intent, GPT-4o-mini for reply
8
🏗️
System Design for LLM Apps
Key architectural decisions:
  • CAP vs Latency: Cosmos CP for chat, AI Search AP for docs
  • Caching: Semantic cache Redis vs Prompt cache OpenAI
  • Queue for Rate Limits: Service Bus on 429. Back-pressure
  • Circuit Breaker: Polly for OpenAI. Fallback to cached response
9
🗄️
Vector DB Strategies
Solving complex retrieval:
  • DiskANN vs HNSW: DiskANN=1B scale SSD. HNSW=<100M RAM fast
  • Cosmos DB 1ms: User memory + real-time. VectorDistance + SQL
  • AI Search 40ms: Full-text + Vector + Facets. Better for search UX
  • Multi-tenancy: Partition /tenantId. DiskANN per partition

🔭
Observability for Agents
Creating transparent & debuggable systems:
  • Structured Logging: Per-agent log `agent=coder, action=tool_call`
  • Distributed Tracing: OpenTelemetry + Microsoft.SemanticKernel*
  • Health Checks: /health checks OpenAI, Cosmos, Search
  • Cost Tracing: Meter.CreateCounter("llm.tokens") per userId
11
🔐
Security & RBAC in Agents
Maintain least privilege gracefully:
  • Entra ID + ClaimsPrincipal: SK auto-injects. user.IsInRole("HR")
  • Tool-level RBAC: [RequiresRole("Admin")] on KernelFunction
  • Audit Logs in Purview: Every tool call logged. DPDP compliance
  • Secretless: Managed Identity for OpenAI, Cosmos. No keys in code
